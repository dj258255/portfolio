Performance

JPA N+1 쿼리 문제 해결

문제 상황

게시판 목록 조회 시 사용자 정보를 가져오는 과정에서 N+1 쿼리가 발생하여 응답 시간이 느려지는 문제

해결 방법

@EntityGraph와 fetch join을 활용하여 한 번의 쿼리로 필요한 데이터를 모두 가져오도록 최적화

JPA Spring Boot Query Optimization
Security

JWT 토큰 만료 처리

문제 상황

JWT 토큰 만료 시 사용자가 갑자기 로그아웃되어 사용성이 떨어지는 문제

해결 방법

Refresh Token을 활용한 자동 토큰 갱신 시스템 구축과 interceptor를 통한 토큰 상태 관리

JWT Spring Security Redis
Database

대용량 데이터 페이징 성능 개선

문제 상황

100만 건 이상의 데이터에서 페이징 처리 시 OFFSET 사용으로 인한 성능 저하

해결 방법

커서 기반 페이징(Cursor-based Pagination)으로 변경하여 일정한 응답 속도 확보

MySQL Pagination Query Performance
Concurrency

동시성 문제와 락 처리

문제 상황

동시에 여러 사용자가 같은 데이터를 수정할 때 발생하는 데이터 불일치 문제

해결 방법

Optimistic Locking과 @Version을 활용하여 동시성 제어 및 충돌 감지 시스템 구현

JPA Concurrency Locking
Memory

메모리 누수 및 GC 튜닝

문제 상황

대용량 파일 업로드 처리 중 메모리 사용량이 급증하여 서버 불안정 현상 발생

해결 방법

스트림 처리와 청크 단위 업로드로 메모리 사용량 최적화, JVM 옵션 튜닝으로 GC 성능 개선

JVM Memory Management File Upload
API

Circuit Breaker 패턴과 Retry 로직을 적용하여 장애 전파 방지 및 자동 복구 시스템 구축
